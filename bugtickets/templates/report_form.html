Using snAPP's reporting system

At snAPP, we prioritise addressing the problems which are being experienced by the most members. This is decided through a voting system.

It's very simple.

If you experience a bug, search existing bug reports.

You can do this by either scrolling through the list of bugs or, more efficiently, search by a keyword (e.g. 'login', 'crash'). Click on the bug's title to find out more details.

Chances are someone else is experiencing the same thing and has reported it already.

If this is the case, 'Upvote' the report and feel free to leave a comment.

If nobody has reported the problem yet, complete a Report Form.

You'll be notified when we have begun to work on fixing the problem, or our administrator may leave a comment seeking more details about your problem.

Report effectively

When reporting a bug, please be as specific as possible. We will sort out the issue quicker if the information provided is both accurate and clear.

Please follow these guidelines:

Title: Provide a short title of your choice which sums up the problem you are experiencing.

Bug type: Select the heading from the dropdown list which you believe your bug falls under:

Functional - If something that you expect snAPP to do is hard, awkward, confusing, or impossible - report it!
Communication - Everything you need to know in order to use snAPP should be available on the screen. If it isn't, report it!
Something Missing - If you cannot save changes, upload an image, access a page or do something you expect to be able to do - report it!.
Spelling & Grammar - you know what these are...
Error Notices - If you experience an error but don't know how to handfle it, report it!
Bad Calculations - when the math (total amount, numbers of tickets or comments or upvotes, etc) just does not seem to be working, report it!
Not Doing What it Should! - If you expect to go from A to B but end up at Z, report it!

Description: Describe your problem in the most concise and accurate way you can. It helps us to solve issues quickly!
